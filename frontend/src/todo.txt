choose from, or all of them: their user profile, and any chat they participated

CONSOLE ERRORS
- There must not be any error in the console, actually. So, the 401 one must be 
gone. For the automatic login, this means receiving a non HttpOnly cookie just 
to indicate the presence of the refresh token cookie, which is hidden from JS.
- As for the other 401 errors, refresh the access token cookie yourself every 4 
minutes instead of waiting for it to expire.
- As for the fact that after 30 days the refresh token expires, which might 
lead to a 401 error as well, it's okay, as it wouldn't show during the defense. 
While these API responses are not errors, and are part of the ordinary function 
of a web app, an evaluator might see them as errors so I'd rather save us the 
headache.
- Before looking into WebSockets, test the app and check the console on all 
three browsers to fix any potential issue.
